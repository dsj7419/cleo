# cleo docs

Documentation synchronization and drift detection.

<Info>**Agent Relevance**: Medium - Used for documentation maintenance</Info>

## Synopsis

```bash
cleo docs <subcommand> [options]
```

## Description

The `docs` command maintains documentation synchronization by detecting drift between scripts and documentation, validating wrapper template completeness, and checking critical commands in README.

## Subcommands

| Subcommand | Description |
|------------|-------------|
| `sync` | Run drift detection (checks scripts vs docs) |
| `gap-check` | Validate knowledge transfer from review docs to canonical docs |

## Sync Options

| Option | Description |
|--------|-------------|
| `--quick` | Quick check (commands only) |
| `--full` | Full check (default) |
| `--strict` | Exit with error on any drift |
| `--json` | JSON output |

## Gap-Check Options

| Option | Description |
|--------|-------------|
| `--epic <id>` | Filter by epic ID |
| `--task <id>` | Filter by task ID |
| `--all-review` | Check all review docs (default) |
| `--json` | JSON output (default) |
| `--human` | Human-readable output |

## Examples

<CodeGroup>

```bash Full Drift Detection
cleo docs sync
```

```bash Quick Check
cleo docs sync --quick
```

```bash Fail on Warnings
cleo docs sync --strict
```

```bash Check All Review Docs
cleo docs gap-check
```

```bash Epic-Specific Check
cleo docs gap-check --epic T2526 --human
```

</CodeGroup>
